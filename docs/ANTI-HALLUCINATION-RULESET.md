# 정확도/무환각 최종 심사자 규칙 설계

## 0. 목적
4개 GPT 결과를 비교·분석·요약할 때 **환각을 최소화**하고 **정확도를 높이는 최종 심사자 규칙**을 정의한다.

## 1. 핵심 원칙
1) **원문 잠금(Knowledge Lock)**: 심사자는 RAW 응답만 사용, 외부 지식 추가 금지  
2) **합의/충돌 분리**: 공통 사실만 합의로 인정  
3) **근거 부재 표시**: 근거 없는 주장은 “가설”로 격리  
4) **논리 정합성 체크**: 내부 모순은 신뢰도 하향  
5) **신뢰도 등급화**: High/Medium/Low로 분류  

## 2. 운영 규칙 (필수)
### 2.1 원문 잠금 규칙
- 심사자는 **에이전트 응답 RAW만 사용**  
- 새 수치/고유명사 추가 금지  
- 외부 지식 보완 금지

### 2.2 합의/충돌 분리 규칙
- **4개 응답에서 공통 언급된 사실만 “합의”**
- 하나라도 다르면 “충돌”로 분류
- 충돌 항목은 최종 요약에서 단정 금지

### 2.3 근거 부재 규칙
- 주장에 근거가 없으면 **“근거 부재” 라벨**
- 근거 부재는 “가설”로만 표시

### 2.4 논리 정합성 규칙
- 동일 에이전트 내 모순 발견 시 **해당 에이전트 신뢰도 하향**
- 모순 항목은 별도 섹션에 격리

### 2.5 신뢰도 등급 규칙
- 합의된 주장: **High**
- 2:2 분할: **Medium**
- 단일 주장: **Low**

## 3. 요약 구성 규칙
1) Executive Summary는 **High/Medium만 반영**
2) Low는 “불확실성/가정” 섹션으로 분리
3) 결론은 **합의된 범위 내에서만** 작성

## 4. 종료 조건 규칙
- 합의 항목이 부족하면 “정보 부족” 명시
- 불확실성 과다 시 “추가 검증 필요” 자동 출력

## 5. 적용 제안
- `Final Synthesis` 프롬프트에 본 규칙 삽입
- “합의/충돌/가설/모순” 테이블을 고정 포맷으로 출력
