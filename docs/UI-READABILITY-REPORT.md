# UI 가독성/레포트 정돈 개선안

## 1) 문제 요약
- 일부 텍스트 대비가 낮아 “안 보임” 이슈 발생
- 레포트 문서(결과 뷰)의 계층/간격/메타 정보 정돈 부족
- 빈 상태/보조 텍스트가 과도하게 흐려져 핵심 안내가 약함

## 2) 가독성 개선안 (우선순위)
### P0: 즉시 개선
- 본문 텍스트 기본 색상: `text-base-900`로 통일
- placeholder/메타 텍스트 대비 상향 (30% → 60% 수준)
- 결과 메타(시간/소스) 텍스트 색 고정, 투명도 최소화
- 빈 상태(Idle) 안내문 대비 상향 (opacity 낮추지 않기)

### P1: 일관성 강화
- 텍스트 스케일 규칙화: 헤더 28/22/18, 본문 16, 보조 12
- 라인 높이 통일: 본문 1.6~1.8
- 버튼/탭 레이블 대비 동일 기준 적용

### P2: 접근성 체크
- 대비 기준 4.5:1 이상 (본문 기준)
- 다크/라이트 대비 동시 검수

## 3) 레포트 문서 정돈 개선안
### A. 구조 정리
- 상단 메타 블록 고정: 제목/시간/활성 에이전트/요약 1줄
- 섹션 헤더 룰: H2/H3 간격 규칙 통일
- 긴 문단은 “소제목 + 2~3문단” 단위 분리

### B. 타이포 정리
- 헤더: `font-display` + 굵기 800 이상
- 본문: `font-sans` + 16px, line-height 1.7
- 강조: 색상 1개만 사용 (teal 또는 amber 중 택1)

### C. 시각 구획
- 섹션 간 24~32px 간격
- 구분선/라인은 연한 회색 + 최소 두께 1px
- 레포트 카드 내 패딩 48~64px로 고정

## 4) UI 컴포넌트별 개선 포인트
- **탭 버튼**: 활성/비활성 대비 차 확대
- **내보내기 버튼**: 아이콘/라벨 색 대비 상향
- **결과 카드**: 제목과 본문 사이 여백 확대
- **목차/헤더**: 섹션 앵커 제공(옵션)

## 5) 구현 제안 (현재 스택 기준)
- Tailwind 토큰 재정의: `text-base-900`, `text-base-800/70`
- `prose` 기본 스타일 오버라이드 (h2/h3/strong/blockquote)
- 결과 뷰 상단에 “메타 블록 컴포넌트” 추가
- 빈 상태 UI의 대비/메시지 강화

## 6) 체크리스트 (QA)

- [ ] 메타 텍스트가 배경 대비 충분
- [ ] 헤더 계층별 크기/굵기 일관
- [ ] 빈 상태 안내문 가독성 확보
