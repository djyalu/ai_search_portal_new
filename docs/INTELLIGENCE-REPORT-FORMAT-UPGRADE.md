# 종합 인텔리전스 보고서 양식 개선안 (NotebookLM 이상 품질 목표)

## 0. 목적
NotebookLM 수준을 넘어서는 **컨설팅/리서치급 인텔리전스 보고서** 포맷을 제안한다.  
핵심은 “읽자마자 핵심이 보이고, 근거·불확실성·의사결정 포인트가 분리되어 있는 구조”다.

## 1. 설계 원칙
1) **결론 선행**: Executive Summary → 근거 → 세부 → 부록 순
2) **불확실성 명시**: 가정/한계/추정치 분리
3) **의사결정 중심**: 액션·우선순위·리스크 노출
4) **신뢰도 표시**: 항목별 Confidence 등급
5) **정보 출처 레이어링**: 내부 관측 / 외부 검증 / 추론 구분

## 2. 최상위 목차(권장)
### 2.0 커버 & 메타
- 제목, 기준 시점, 대상 범위, 작성 목적

### 2.1 Executive Summary (3~5줄)
- 핵심 결론 3줄 + 리스크 1줄 + 다음 액션 1줄

### 2.2 핵심 인사이트 Top 5
- 인사이트 / 근거 요약 / 영향도 / 신뢰도

### 2.3 합의 vs 불일치 매트릭스
- 에이전트별 공통·충돌 관점 분리

### 2.4 구조적 원인-경로-결과
- 원인(Drivers) → 메커니즘(Path) → 결과(Impact)

### 2.5 시나리오 플래닝
- 베이스/상승/하락 시나리오 + 트리거 조건

### 2.6 리스크 & 불확실성
- 데이터 공백 / 논리 충돌 / 외부 변수

### 2.7 전략 옵션 & 권고 액션
- 옵션 A/B/C, 기대효과, 리스크, 비용/난이도

### 2.8 모니터링 체크리스트
- KPI, 관찰 포인트, 업데이트 주기

### 2.9 부록
- 에이전트 스냅샷, 원문 요약, 용어 정의

## 3. 표준 섹션 포맷 (템플릿)

### 3.1 Executive Summary (템플릿)
- 결론: …
- 리스크: …
- 다음 액션: …

### 3.2 인사이트 카드 (템플릿)
| 항목 | 내용 |
|---|---|
| 인사이트 | … |
| 근거 요약 | … |
| 영향도 | High / Medium / Low |
| 신뢰도 | ★★★★★ |

### 3.3 시나리오 표
| 시나리오 | 가능성 | 핵심 조건 | 영향 |
|---|---|---|---|
| 베이스 | 중 | … | … |
| 상승 | 중~고 | … | … |
| 하락 | 중 | … | … |

## 4. 시각화/구조 개선 포인트
- **Key Takeaways 카드화**: 3~5개 카드 형태로 시각 강조
- **합의/불일치 테이블 고정**: 보고서마다 동일 위치에 배치
- **리스크/불확실성 분리**: 혼합 금지
- **Confidence 표시**: 각 핵심 주장에 별점 또는 구간화(High/Med/Low)

## 5. 문체/톤 가이드
- 분석 톤: 컨설팅 보고서 스타일, 간결하고 단정
- 불확실성은 “~일 가능성”으로 명시
- 감성적 수사 최소화, 구조적 문장 유지

## 6. NotebookLM 대비 차별화 포인트
- **결론 선행 + 액션/리스크 분리**
- **에이전트 합의·충돌의 구조적 분해**
- **시나리오/트리거/모니터링 통합**
- **신뢰도 등급으로 의사결정 가이드 강화**

## 7. 적용 제안
- `Final Synthesis` 프롬프트에 위 목차 구조 반영
- 섹션별 토큰 예산 분리(예: Executive 10%, Insight 20%, Scenario 20%)
- 불확실성/가정 섹션은 항상 강제 출력
