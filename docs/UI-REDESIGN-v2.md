# UI 전면 개편안 v2 (가독성/설정 화면 보강)

## 0) 문제 정의 (사용자 피드백 기반)
- 글자가 안 보임 → 대비(contrast) 부족, 텍스트 계층/크기 불명확
- 설정 화면 부재 → 시스템/계정/모델/내보내기 설정 접근성 부족
- 결과 비교의 신뢰성 정보가 약함 → 지표/메타정보 재배치 필요

## 1) 핵심 목표
- **가독성 우선**: 텍스트 대비 7:1 수준으로 설계
- **설정 접근성 강화**: 언제든 상단에서 1클릭 진입
- **비교 신뢰도 강화**: 근거/응답시간/불일치 지표를 상시 노출

## 2) 네비게이션 구조 (재설계)
- 좌측 고정 내비 + 상단 유틸 바(검색/새 분석/설정)
- 메뉴: Dashboard / New Analysis / Compare / History / Exports / Settings
- Settings는 전역 접근(상단 아이콘 + 좌측 메뉴 동시 제공)

## 3) 레이아웃 (가독성 최우선)
### A. New Analysis
- 좌측: 입력 패널(제목/태그/목표 + 프롬프트)
- 우측: 실행 옵션(모델 선택/라운드/검증)
- 하단: 실행 큐 + 진행률 타임라인

### B. Results
- 상단: 요약 헤더(합의 지수/불일치/응답 시간)
- 중앙: 모델별 결과 카드(문단별 강조 레이어)
- 우측: 근거 리스트/링크/출처 메타 패널

### C. Settings (신규/강화)
- 섹션: 계정/세션, 모델/API, 자동 저장, Export(Notion/Markdown), 테마/가독성, 알림
- 고급: 프롬프트 템플릿, 검증 라운드 프리셋

## 4) 타이포그래피 & 가독성 스펙
- 기본 본문 16px 이상, 라인높이 1.6
- 헤더 계층: 28/22/18
- 본문 색상: #111 (밝은 배경) / #F5F5F5 (어두운 배경)
- 링크/강조: 대비 4.5:1 이상
- 긴 텍스트는 문단 경계 강조(미세 구분선)

## 5) 색상 팔레트 (고대비)
- 배경: Warm Paper (#F7F4EF) + Ink (#111111)
- 강조: Teal (#13B5B1), Amber (#F0A202), Coral (#F25C54)
- 카드 배경: #FFFFFF (shadow + 1px border)
- 접근성: 텍스트 대비 7:1 기준

## 6) 컴포넌트 가이드
- 결과 카드: 상단 메타(모델/응답시간/근거 수)
- 비교 하이라이트: 문장 단위 배경 강조
- 타임라인: 상태 색 + 진행 뱃지
- 설정: 섹션별 스위치/토글 + 저장 상태 표시

## 7) 모션/상태
- 로딩: 스켈레톤 + 진행률 바
- 결과 업데이트: 카드 페이드 인(120ms)
- 에러: 고대비 배너 + 재시도 버튼

## 8) 모바일
- 상단 탭 내비 + 설정 아이콘 유지
- 결과는 세로 카드 스택
- 비교는 스와이프 전환

## 9) 구현 체크리스트 (현재 스택 기준)
- Tailwind 토큰: 색/폰트/그림자/라인높이 확장
- Settings 라우트 추가 + UI 컴포넌트
- 결과 카드/메타/근거 패널 분리 컴포넌트화
- 대비 자동 점검(간단한 UI 테스트 스냅샷)
