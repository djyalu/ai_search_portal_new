# UI 개선안 v3 (원래 기능 유지 + 가독성 강화 + 레포트 정돈)

## 0) 목표
- **원래 기능 100% 유지**: 에이전트 토글, 실시간 스트리밍, 타임라인, 히스토리, 내보내기
- **가독성 개선**: 텍스트 대비/계층/간격 개선
- **레포트 정돈**: 메타/구조/섹션 규칙 정립

## 1) 유지할 핵심 기능 (변경 없음)
- 에이전트 ON/OFF 토글
- 실시간 스트리밍 카드(Perplexity/ChatGPT/Gemini/Claude)
- RALPH 단계/타임라인
- 결과 탭(통합/개별/검증)
- Export (PDF/HTML/MD)
- 히스토리 선택/삭제

## 2) 가독성 개선안 (UI 규칙)
### A. 텍스트 대비/색상
- 본문 기본: `#1f2a44` 수준의 짙은 잉크 컬러 고정
- 보조 텍스트: 60~70% 명도 (투명도 남발 금지)
- placeholder: 50~60% 명도 (기존 30% → 상향)

### B. 타이포 계층
- 헤더: 28 / 22 / 18 (font-weight 800+)
- 본문: 16 (line-height 1.6~1.8)
- 보조: 12~13

### C. 레이아웃 간격
- 카드 내부 패딩 48~64px
- 섹션 간격 24~32px
- 표/리스트는 8~12px 단위로 리듬 부여

## 3) 레포트 문서 정돈 (결과 탭)
### A. 상단 메타 블록 (고정)
- 제목 / 생성 시각 / 활성 에이전트 / 요약 1줄
- 메타 블록 배경을 연한 회색(1px border)으로 고정

### B. 섹션 규칙
- H2는 문단 시작 전 24px 여백
- H3는 16px 여백
- HR 사용은 섹션 구분에만 제한

### C. 본문 스타일
- 본문 문단은 2~3문장 단위로 짧게
- 리스트는 bullet spacing 8px
- 강조색 1개만 사용 (Teal 또는 Amber)

### D. 개별 탭 정리
- **통합(Optimal)**: 요약 → 핵심 인사이트 → 근거 → 결론
- **개별(Individual)**: 모델명 고정 헤더 + 카드 간 간격 일관
- **검증(Report)**: 검증 항목/점수/근거를 표 형태로 우선 배치

## 4) 화면별 개선안
### A. 입력 영역
- 입력 텍스트 대비 상향
- 실행 버튼 크기 유지, 텍스트 대비 강화
- 에이전트 토글 ON/OFF 배지 유지

### B. 결과 영역
- 탭 버튼 활성/비활성 대비 명확화
- 레포트 본문 `prose` 규칙 재정의
- 빈 상태 문구 대비 상향(안내 메시지 강조)

### C. 사이드 패널
- 타임라인 텍스트 대비 상향
- 진행 바/상태 칩 강조색 통일

## 5) 구현 제안 (현재 코드 기준)
### A. 스타일 토큰 정리
- 기본 텍스트 컬러 상향 (base-900 기준)
- 보조 텍스트는 opacity 대신 고정 색상 사용

### B. 레포트 전용 컴포넌트
- `ReportMeta` (메타 정보)
- `ReportSection` (섹션 규칙)
- `ReportTable` (검증 항목 정리)

### C. Tailwind 확장
- `prose` 세부 스타일 override
- `text-base-800/60` 같은 불명확한 투명도 최소화

## 6) 적용 우선순위 (Phase)
1. 가독성 개선 (텍스트 대비 + 빈 상태 + 메타)
2. 레포트 구조 정돈 (메타 블록 + 섹션 규칙)
3. 검증 리포트 표/카드형 구조화
